# Paper Directory

## Files

| File | Purpose |
|------|---------|
| `draft.md` | Working prose draft (primary authoring document) |
| `main.tex` | LaTeX manuscript skeleton (for submission) |
| `definitions.md` | Formal mathematical definitions |
| `proofs.md` | Proven propositions with complete proofs |
| `theorem_sketches.md` | Conjectures and research directions (status-labeled) |
| `claim_ledger.md` | Controls allowed claim strength (proven / synthetic / conjectural / not claimed) |
| `literature.md` | Citation scaffold — entries marked VERIFY must be checked before use |
| `outline.md` | Paper section outline |
| `notes.md` | Research notes and open questions |

## Workflow

1. Write and iterate in `draft.md`
2. Check claims against `claim_ledger.md` before adding to LaTeX
3. Verify citations in `literature.md` before including in `main.tex`
4. Proven results go in `proofs.md`; conjectures stay in `theorem_sketches.md`
5. Compile LaTeX (see below)

## Build Instructions

From repo root:

```bash
# Regenerate figures (required before first compile)
python experiments/generate_figures.py

# Check all assets are present
python scripts/check_paper_assets.py

# Compile LaTeX
cd paper
pdflatex -interaction=nonstopmode main.tex
```

**Note:** `refs.bib` is not finalized yet. Bibliography will not fully resolve until citation verification is complete (see `literature.md` VERIFY entries). The paper compiles with warnings but no errors.

## Anticipated Reviewer Concerns

**Q: Is this just bounded ZNE repackaged?**
No. Bounded ZNE proposes an estimator. This paper studies the *identifiability structure* of ZNE as an inverse problem. The ambiguity diameter is a property of the inference problem, not of any particular estimator.

**Q: Does this defeat QEM lower bounds?**
No. We explicitly state this in the Introduction, Discussion, and Limitations. The lower bounds apply to generic worst-case circuits. We study when structured assumptions (function class + spectral bounds) affect identifiability, not whether they circumvent complexity results.

**Q: Is there hardware evidence?**
No. All results are synthetic. This is stated in the abstract, experiments section, and limitations. Hardware validation is explicitly listed as future work.

**Q: Is ambiguity the same as estimation error?**
No. Ambiguity diameter measures the range of *consistent* values, not the error of any estimator. Small ambiguity means the problem is well-posed; it does not guarantee that any particular estimator achieves small error. The bias-ambiguity tradeoff (§5.5) makes this distinction explicit.
